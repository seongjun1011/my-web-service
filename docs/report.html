1. 프로젝트 개요

1.1 배경 및 목적

현대인의 바쁜 생활 속에서 냉장고 속 식재료를 체계적으로 관리하기 어렵고, 유통기한이 지난 식재료를 버리는 경우가 빈번하다. 본 프로젝트는 AI 기술을 활용하여 식재료를 자동으로 인식·등록하고, 유통기한 임박 알림 및 보유 재료 기반 레시피 추천을 제공하는 스마트 저장고 관리 서비스 SmartPantry를 개발하였다.

1.2 주요 목표

  • 카메라 촬영으로 식재료를 자동 인식하여 냉장고에 등록
  • 영수증 OCR을 통한 구매 식재료 일괄 등록
  • 유통기한 D-Day 시각화 및 임박 재료 푸시 알림
  • 보유 재료 기반 AI 레시피 추천
  • 관리자 대시보드를 통한 서비스 운영 관리

1.3 프로젝트 범위

7. 주요 기능 구현

7.1 소셜 로그인 (OAuth 2.0)

Passport.js를 통해 카카오/Google OAuth 2.0 인증을 구현하였다. 로그인 성공 시 소셜 ID를 기준으로 DB에서 사용자를 조회하며, 최초 로그인의 경우 자동 가입 처리 후 약관 동의 화면으로 이동한다. 세션은 express-session을 통해 서버 메모리에 관리되며, 쿠키 기반으로 인증 상태를 유지한다.

7.2 식재료 스캔

사용자가 카메라로 식재료를 촬영하면 canvas API를 통해 이미지를 JPEG base64로 인코딩하여 서버로 전송한다. 서버는 이를 AI 서버로 전달하며, AI 서버에서는 모드에 따라 YOLO 또는 LLM OCR 파이프라인을 실행한다. 결과로 반환된 식재료 목록은 스캔 결과 화면에서 개별 선택 후 저장고에 등록 가능하다.

7.3 유통기한 D-Day 시각화

내 냉장고 화면에서 각 식재료의 D-Day를 실시간으로 계산하여 표시한다. 남은 일수에 따라 HSL 색상 스펙트럼(빨강~초록, 14일 기준)을 적용하여 시각적으로 긴박도를 전달한다. 홈 화면에는 D-3 이내 임박 재료를 별도로 강조 표시한다.

7.4 Web Push 알림

VAPID(Voluntary Application Server Identification) 키 기반 Web Push 표준을 구현하였다. 사용자가 알림을 허용하면 브라우저의 PushManager가 구독 객체를 생성하고, 서버 DB의 push_subscriptions 테이블에 저장된다. node-cron을 통해 매일 오전 9시(Asia/Seoul)에 유통기한 D-1~3인 식재료를 보유한 사용자에게 자동으로 알림을 발송한다. 만료된 구독(410/404 응답)은 자동으로 DB에서 삭제한다.

7.5 관리자 시스템

DB의 is_admin 컬럼으로 관리자를 구분하며, 서버 미들웨어(isAdmin)에서 모든 관리자 API 요청을 검증한다. 프론트엔드에서도 is_admin 값에 따라 하단 네비게이션 아이콘을 교체하여 관리자 대시보드 진입을 제공한다. 관리자 대시보드는 통계/유저관리/공지관리 3개 탭으로 구성된다.

7.6 DB 커넥션 풀

초기에는 단일 MySQL 커넥션을 사용하였으나, MySQL의 기본 wait_timeout(8시간) 이후 커넥션이 자동 종료되어 서비스 장애가 발생하는 문제가 있었다. mysql2의 createPool로 교체하여 자동 재연결 및 최대 10개 동시 커넥션을 지원하도록 개선하였다.
